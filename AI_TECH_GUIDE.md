# MyWallet — AI 기술 상세 가이드

## 이 앱에서 사용한 AI 기술

---

## 1. LLM (Large Language Model) — OpenAI GPT-4o mini

### 무엇인가?
대규모 텍스트 데이터로 학습된 언어 모델. 사람처럼 자연스러운 문장을 이해하고 생성합니다.

### 이 앱에서 하는 일
사용자의 소비 데이터를 분석해서 **충동소비 원인과 맞춤 코칭**을 한국어로 생성합니다.

```
입력: 총 지출 1,240,000원 / 충동소비 의심 18% / 주요 카테고리: 배달(340,000원), 카페(120,000원)
출력:
  원인: 야간 시간대 배달 앱 결제가 집중되어 피로·허기 상태에서의 충동적 소비 패턴이 보입니다.
  코칭: 21시 이후 배달 앱 알림을 끄고, 간단한 야식 재료를 미리 준비해두는 것이 효과적입니다.
```

### 선택 이유
| 항목 | 내용 |
|------|------|
| 한국어 품질 | GPT-4o mini는 한국어 자연어 생성 품질이 높음 |
| 비용 | 1회 분석당 약 $0.0002 이하 (매우 저렴) |
| 응답 속도 | mini 모델로 빠른 응답 |
| API | OpenAI Python SDK로 간단 연동 |

## 2. RAG (Retrieval-Augmented Generation) — ChromaDB + Embedding

### 무엇인가?
RAG는 **검색(Retrieval) + 생성(Generation)** 을 결합한 AI 기법입니다.
LLM이 학습된 일반 지식만으로 답하는 것이 아니라, **미리 준비한 도메인 전문 문서를 검색해서 GPT 프롬프트에 함께 넣어주는** 방식입니다.

### 왜 RAG가 필요한가?
GPT만 쓰면 이런 뻔한 답이 나옵니다:
> "충동소비를 줄이려면 소비 일지를 작성하세요."

RAG를 쓰면 직접 작성한 전문 코칭 문서 기반으로 구체적인 답이 나옵니다:
> "야간 배달 충동소비는 주로 피로·보상 심리에서 발생합니다. 21시 알림 차단 + 대체 행동(따뜻한 음료 준비)을 병행하면 3주 내 개선 효과가 있습니다."

### RAG 파이프라인

```
[코칭 문서 6개 — data/knowledge/]
  stress_spending.md   → 스트레스성 충동소비 대응법
  night_shopping.md    → 야간 충동 구매 패턴 분석
  delivery_food.md     → 배달 과소비 원인과 해결
  online_shopping.md   → 온라인 쇼핑 충동 구매 차단
  weekend_spending.md  → 주말 과소비 패턴
  cafe_repeat.md       → 카페 반복 지출 관리
         │
         ▼
[임베딩 — OpenAI text-embedding-3-small]
  문서를 숫자 벡터로 변환 (1536차원)
  예) "스트레스" → [0.23, -0.11, 0.87, ...]
         │
         ▼
[벡터 DB 저장 — ChromaDB]
  로컬 파일(data/.chromadb/)에 벡터 인덱스 저장
  최초 1회만 인덱싱, 이후 재사용
         │
         ▼
[쿼리 생성 — 사용자 소비 패턴]
  "충동소비 카테고리: 배달(340,000원), 야간지출비율: 28.3%"
         │
         ▼
[유사도 검색 — Cosine Similarity]
  쿼리 벡터 ↔ 문서 벡터 거리 계산
  가장 유사한 2개 섹션 반환
         │
         ▼
[GPT 프롬프트에 주입]
  소비 데이터 + RAG 검색 결과 → GPT-4o mini → 원인/코칭 출력
```

### 임베딩이란?
텍스트를 숫자 벡터로 변환하는 기술입니다.
의미가 비슷한 문장은 벡터 공간에서 가까이 위치합니다.

```
"야간에 배달 자주 시킴"  →  벡터 [0.23, 0.87, ...]
"밤에 음식 주문이 많음"  →  벡터 [0.21, 0.85, ...]  ← 거리 가까움
"주말에 쇼핑 과소비"    →  벡터 [0.71, 0.12, ...]  ← 거리 멂
```

## 3. 규칙 기반 충동소비 탐지 (Rule-based Detection)

AI 모델은 아니지만, 데이터 분석 로직으로 충동소비를 자동 분류합니다.

| 플래그 | 기준 | 의미 |
|--------|------|------|
| `flag_over_cat_avg` | 카테고리 평균의 N배 초과 | 평소보다 이 카테고리에서 갑자기 많이 씀 |
| `flag_night` | N시 이후 결제 | 야간 감정적 구매 가능성 |
| `flag_freq_impulse` | 동일 카테고리 하루 N건+ | 같은 곳에서 반복 결제 |
| `flag_over_daily_avg` | 하루 지출 일평균 N배 초과 | 오늘 유독 많이 쓴 날 |
| `flag_weekend_night` | 주말 + 야간 결제 | 충동소비 고위험 구간 |

## 4. 전체 AI 파이프라인 요약

```
[사용자 소비 데이터 (CSV/Excel)]
         │
         ▼
[규칙 기반 충동소비 탐지]  ← 5가지 플래그 자동 분류
         │
         ▼
[소비 요약 생성]
  총지출, 충동소비 비율, 야간 비율, 카테고리별 금액
         │
         ├──→ [RAG 쿼리 생성]
         │           │
         │           ▼
         │    [ChromaDB 유사도 검색]
         │    text-embedding-3-small로 코칭 문서 검색
         │           │
         ▼           ▼
    [GPT-4o mini 프롬프트 구성]
    소비 데이터 + RAG 코칭 문서 2개
         │
         ▼
    [출력: 원인 한 문장 + 코칭 한 문장]
```

## 5. 배포 방법 (Streamlit Cloud)

### 배포하면 실제로 인터넷에서 접근 가능한 URL이 생깁니다.

```
1. share.streamlit.io 접속
2. GitHub 계정 연동
3. Repository: meene11/my_wallet_pattern
   Branch: main
   Main file: app.py
4. Secrets 탭에서 환경변수 등록:
   OPENAI_API_KEY = "sk-..."
5. Deploy 클릭
```

### 배포 후 구조
```
인터넷 사용자
     │  https://your-app.streamlit.app
     ▼
Streamlit Cloud 서버
     │  app.py 실행
     ▼
OpenAI API 호출 (AI 코칭)
ChromaDB (로컬 벡터 인덱스)
```

### 주의사항
- ChromaDB는 로컬 파일 기반 → 배포 시 `data/knowledge/` 폴더가 GitHub에 있어야 함
- `.env`는 git에 올리지 말고 Streamlit Cloud Secrets에 직접 입력
- 무료 플랜으로 충분 (소규모 개인 앱)
