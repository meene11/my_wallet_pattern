# MyWallet — 개인 소비 분석 AI 프로젝트 구조서

## 프로젝트 개요

카드/지출 내역 파일을 업로드하면 소비 패턴을 분석하고, 충동소비를 자동 탐지하며,
AI가 원인 분석과 맞춤 코칭을 제공하는 Streamlit 웹 앱.

---

## 기술 스택

| 영역 | 기술 |
|------|------|
| 웹 프레임워크 | Streamlit |
| 데이터 처리 | Pandas, NumPy |
| 시각화 | Matplotlib, Seaborn, Plotly |
| AI 코칭 | OpenAI GPT-4o mini |
| 환경 변수 | python-dotenv |
| 언어 | Python 3.x |

## 데이터 파이프라인

```
[파일 업로드 / 샘플 데이터]
         │
         ▼
[load_file()]
 CSV(UTF-8/EUC-KR) 또는 Excel 파싱
         │
         ▼
[auto_map_columns()]
 컬럼명 키워드 매칭 → 날짜/금액/카테고리 자동 인식
 실패 시 데이터 타입 추론으로 보완
         │
         ▼
[preprocess()]
 ├── 날짜/시간 파싱
 ├── 금액 정제 (쉼표·원 제거 → 숫자 변환)
 ├── 카테고리 자동 분류 (가맹점명 → 업종 매핑 80+ 키워드)
 ├── 파생 컬럼 생성 (요일, 시간대, 주차)
 └── 충동소비 탐지 (5가지 규칙 기반 플래그)
         │
         ▼
[get_summary()]
 총지출 / 건수 / 일평균 / 충동소비 합계 등 집계
         │
         ▼
[Streamlit UI 렌더링]
 탭1: 전체 요약   탭2: 패턴 분석
 탭3: 충동소비 탐지 + AI 코칭   탭4: 원본 데이터
```

## 주요 기능 상세

### 1. 데이터 로드 (`data_loader.py`)

- **다중 인코딩 지원**: UTF-8, EUC-KR, Excel 자동 감지
- **컬럼 자동 매핑**: 카카오페이·신한카드·직접작성 등 다양한 형식 대응
  - 1단계: 컬럼명 키워드 매칭 (80+ 후보어)
  - 2단계: 데이터 타입 추론 (날짜 파싱 성공률, 숫자 분포, 고유값 수)
- **가맹점 자동 분류**: 80+ 가맹점명 → 14개 카테고리 (배달, 카페, 편의점 등)

### 2. 충동소비 탐지 (`data_loader.py`)

5가지 규칙 기반 플래그:

| 플래그 | 기준 |
|--------|------|
| `flag_over_cat_avg` | 해당 카테고리 평균의 2배 초과 |
| `flag_night` | 21시 이후 결제 |
| `flag_freq_impulse` | 같은 날 동일 카테고리 3건 이상 |
| `flag_over_daily_avg` | 하루 지출이 개인 일평균의 1.5배 초과 |
| `flag_weekend_night` | 토·일 21시 이후 결제 |

### 3. 시각화 (`app.py`)

- **탭1**: 카테고리 파이차트 + 범례 테이블 + 일별 추이 (7일 이동평균)
- **탭2**: 요일별 막대차트 / 시간대별 막대차트 / 카테고리×요일 히트맵
- **탭3**: 충동소비 거래 목록 / 탐지이유별 건수 / 카테고리별 금액 (스크롤 대응) / 소비 진단
- **탭4**: 필터링 + CSV 다운로드

### 4. AI 소비 코치 (`gemini_analyzer.py`)

- **모델**: GPT-4o mini
- **입력**: 총지출, 충동소비 비율, 야간지출, 상위 카테고리 5개, 충동소비 카테고리 3개
- **출력**: `원인:` 한 문장 + `코칭:` 한 문장
- **프롬프트 규칙**: 실제 데이터에 없는 카테고리/상품명 언급 금지
- **캐시 전략**: 데이터 해시(총지출+건수+충동건수) 기반 자동 무효화
- **탭 점프 방지**: `@st.fragment`로 AI 버튼 격리

## 고도화 제안

### Phase 2 — 감정/맥락 분석
- 메모 텍스트에서 감정 키워드 추출 ("스트레스", "피곤해서", "기분 전환")
- 감정 상태 → 충동소비 상관관계 시각화
- 현재 가맹점명 키워드 매핑으로 하드코딩된 부분을 NLP로 대체

### Phase 3 — RAG 소비 코칭
- `data/knowledge/` 에 소비습관 가이드 문서 20~30개 작성
- ChromaDB 벡터DB에 인덱싱
- 유저 패턴과 유사한 코칭 사례 검색 → GPT에 함께 전달
- 더 정확하고 맥락 있는 코칭 가능

### Phase 4 — 예측 및 목표 관리
- 다음 달 지출 예측 (Prophet 시계열 모델, notebooks/04_forecasting.ipynb 활용)
- 카테고리별 예산 설정 + 초과 알림
- 주간/월간 리포트 자동 생성

### Phase 5 — 멀티유저 / 배포
- 사용자 인증 (로그인) + 지출 이력 DB 저장 (PostgreSQL)
- Streamlit Cloud 또는 Docker + EC2 배포
- 카카오페이·토스 API 직접 연동 (파일 업로드 없이 자동 수집)
