# MyWallet — 프로젝트 구조 및 기술 문서

## 프로젝트 개요

카드/지출 내역 파일을 업로드하면 소비 패턴을 분석하고, 충동소비를 자동 탐지하며,
AI가 원인 분석과 맞춤 코칭을 제공하는 Streamlit 웹 앱.

단순 집계가 아닌 **"왜 충동소비를 했는가"** 를 데이터와 AI로 파악하고,
구체적인 행동 변화 조언까지 제공하는 것이 핵심 목적.

## 전체 데이터 파이프라인

```
[파일 업로드 / 샘플 데이터]
         │
         ▼
[카드사 프리셋 적용]  ── 카카오페이 / 신한 / 국민 / 하나 / 자동감지
         │
         ▼
[load_file()]
 CSV(UTF-8/EUC-KR) 또는 Excel 파싱
         │
         ▼
[auto_map_columns()]
 컬럼명 키워드 매칭 → 날짜/금액/카테고리 자동 인식
 실패 시 데이터 타입 추론으로 보완
         │
         ▼
[preprocess()]  ── 사이드바 슬라이더 값 파라미터로 수신
 ├── 날짜/시간 파싱
 ├── 금액 정제 (쉼표·원 제거 → 숫자 변환)
 ├── 카테고리 자동 분류 (가맹점명 → 업종, 80+ 키워드)
 ├── 파생 컬럼 생성 (요일, 시간대, 주차)
 └── 충동소비 탐지 (5가지 규칙, 사용자 조정 가능)
         │
         ▼
[get_summary()]
 총지출 / 건수 / 일평균 / 충동소비 집계
         │
         ▼
[Streamlit UI 렌더링 — 5개 탭]
 ├── 탭1: 전체 요약
 ├── 탭2: 패턴 분석
 ├── 탭3: 충동소비 탐지 + AI 코치
 ├── 탭4: 원본 데이터
 └── 탭5: 예산 관리
```

## AI 코칭 파이프라인 (RAG + LLM)

```
[유저 소비 데이터]
         │
         ├── 소비 요약 (총지출, 충동비율, 야간비율, 상위 카테고리)
         └── 충동소비 메모 감정 텍스트 추출
                   │
                   ▼
[RAG 쿼리 생성]
 "충동소비 카테고리: 쇼핑, 배달 / 메모: 스트레스 받아서 / 야간비율: 32%"
                   │
                   ▼
[ChromaDB 유사도 검색]
 data/knowledge/ 6개 문서 → 가장 유사한 2개 섹션 반환
                   │
                   ▼
[GPT-4o mini 프롬프트 구성]
 소비 데이터 + 감정 메모 + RAG 검색 결과
                   │
                   ▼
[출력]
 원인: (감정/패턴 기반 한 문장)
 코칭: (RAG 문서 참고한 구체적 조언 한 문장)
```

## 주요 기능 상세

### 1. 데이터 로드 (`data_loader.py`)

**무엇을 하는가:** 어떤 형식의 파일이든 자동으로 읽어서 분석 가능한 형태로 변환
**왜 하는가:** 카드사마다 컬럼명이 다르고 인코딩도 달라서 자동화가 필수

- 다중 인코딩: UTF-8, EUC-KR, Excel 자동 감지
- 2단계 컬럼 매핑: 키워드 → 데이터 타입 추론
- 가맹점명 → 14개 카테고리 자동 분류 (80+ 규칙)

### 2. 카드사 프리셋 (`app.py`)

**무엇을 하는가:** 카드사 선택 시 컬럼명 자동 세팅
**왜 하는가:** 매번 수동으로 컬럼 매핑하는 불편함 해소. 카드사별 고유 컬럼명 구조가 다름

- 카카오페이 / 신한카드 / 국민카드 / 하나카드 프리셋 내장
- 자동감지와 병행 사용 (프리셋 우선 적용)

### 3. 충동소비 탐지 + 임계값 조정 (`data_loader.py` + `app.py`)

**무엇을 하는가:** 5가지 기준으로 충동소비 거래를 자동 분류하고 사용자가 기준을 조정
**왜 하는가:** 사람마다 다른 소비 패턴 — 야행성인 사람은 야간 기준을 높여야 정확도 향상

| 플래그 | 기준 | 조정 범위 |
|--------|------|----------|
| `flag_over_cat_avg` | 카테고리 평균 배수 초과 | 1.5~5.0x |
| `flag_night` | 야간 시간 이후 결제 | 18~23시 |
| `flag_freq_impulse` | 동일 카테고리 일일 건수 | 2~10건 |
| `flag_over_daily_avg` | 하루 지출 평균 배수 초과 | 1.2~3.0x |
| `flag_weekend_night` | 주말 야간 결제 | 자동 |

### 4. 메모 감정 분석 (`gemini_analyzer.py`)

**무엇을 하는가:** 충동소비 거래의 메모 텍스트를 AI 프롬프트에 포함
**왜 하는가:** "스트레스 받아서 질러버림" 같은 메모가 있으면 원인 분석이 카테고리 수준을 넘어 감정/상황 수준으로 정밀해짐

- 충동소비 거래 중 메모가 있는 것만 추출 (최대 8개)
- 프롬프트 [충동소비 시 감정/상황] 필드에 포함
- GPT 규칙: "메모에 감정 표현이 있다면 반드시 원인 분석에 반영"

### 5. RAG 소비 코칭 (`rag_engine.py`)

**무엇을 하는가:** 유저 소비 패턴과 가장 유사한 코칭 문서를 검색해서 GPT에 제공
**왜 하는가:** GPT가 일반 지식만으로 답하면 뻔한 조언이 나옴. 직접 작성한 소비 코칭 문서를 참고시키면 더 구체적이고 실용적인 조언 가능

- ChromaDB PersistentClient (로컬 저장)
- 임베딩: OpenAI text-embedding-3-small
- 6개 코칭 문서, 섹션(##) 단위 청킹
- 최초 실행 시 자동 인덱싱 → 이후 재사용

### 6. 예산 관리 탭 (`app.py`)

**무엇을 하는가:** 카테고리별 월 예산 입력 → 실제 지출과 비교 → 초과 시 경고
**왜 하는가:** 분석은 과거를 보여주지만 예산은 미래를 바꿈. 실제 행동 변화로 이어지는 핵심 기능

- 카테고리별 예산 number_input (session_state 유지)
- Progress bar + 메트릭으로 사용/예산 시각화
- 초과 카테고리 빨간 경고 / 정상 초록 표시

## 브랜치별 작업 내역

| 브랜치 | 주요 변경 | 작업 이유 |
|--------|-----------|-----------|
| `feature/gemini-analyzer` | OpenAI GPT-4o mini 연동, AI 코칭 탭3 추가 | Gemini 무료 티어 한국 리전 미지원으로 OpenAI로 전환 |
| `feature/sidebar-thresholds` | 충동소비 임계값 4개 슬라이더 + preprocess() 파라미터화 | 하드코딩된 기준값이 사용자마다 맞지 않는 문제 해결 |
| `feature/card-preset` | 카드사 선택 드롭다운 + 컬럼 오버라이드 | 매번 수동 매핑 불편함 해소 |
| `feature/memo-emotion` | 충동소비 메모 감정 추출 → 프롬프트 주입 | 카테고리 수준 분석 → 감정/상황 수준 분석으로 정밀화 |
| `feature/budget-alert` | 예산 탭5 추가, progress bar + 경고 | 분석 → 행동 변화 연결 고리 |
| `feature/rag-coaching` | RAG 엔진, 6개 코칭 문서, ChromaDB 인덱싱 | GPT 일반 지식 → 도메인 특화 코칭 문서 기반 정밀 답변 |
